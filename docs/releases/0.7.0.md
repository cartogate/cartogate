# Cartogate 0.7.0 — the gate learns to navigate

**Cartogate now checks an agent's work in the browser, not just in the code.** This release extends
the local-first, deterministic gate from "did you re-implement something / break a contract?" to
"did you actually reach the state you claim you reached?" — and hardens the definition of *done*
with enforceable verification contracts and a tamper-evident record of every attestation.

Everything still runs entirely on your machine. No cloud, no network calls, no language model of
its own. Every answer traces back to a line of source or a recorded fact.

---

## The headline: navigation maps (`cartogate.nav`)

Coding agents are good at writing browser scripts and bad at knowing whether the script did what
they said it did. When the thing they need to click has no accessible name, a weak agent will
click *something*, screenshot it, and sign off — a **false completion**. `cartogate.nav` closes
that gap the same way the code gate does: with declared, extracted facts.

A **navigation map** is a checked-in JSON file describing an app's states, the landmarks that
identify each one, and the affordances that move between them. An agent drives the app *through*
the map — reaching a state by construction and verifying it against live landmarks — instead of
guessing from the DOM.

- **Deterministic Navigator** — no exploration; follows declared transitions and confirms a state
  only when its landmarks are actually on the page. A state it can't verify is `LOST`, never
  silently accepted.
- **The extract → seed → crawl → check loop** — `cartogate navmap` seeds a draft map from
  extracted routes (Next.js `app/`/`pages/`, React Router, Vue Router); a live crawl verifies your
  declared states and *proposes* new landmarks (quarantined, never auto-merged); `cartogate nav
  check` traverses a flow deterministically as a CI gate.
- **Bounded frontier discovery** (`nav crawl --discover`) — proposes new states beyond the map
  under five hard controls: non-GET requests aborted in flight, loopback-only (redirects off your
  machine are dead-ends, no override), navigation-semantic clicks only, explicit budgets, and
  quarantined output.
- **Two driver adapters** — a Playwright reference adapter and a Selenium adapter (proving the
  driver seam is honest), plus an in-memory `FakeDriver` so you can unit-test your own maps with no
  browser. Live E2E on both adapters runs on every PR.

**Measured value.** In a blind A/B across four models (two providers, weak and strong tiers), the
map arm produced **zero false completions in every cell** — versus a 100% false-completion rate for
a weak, role-based agent on a task whose target has no accessible name — at **roughly half the tool
calls and cost**, even where the vanilla agent eventually succeeded. (n=5 per cell, one task, our
own visualization; see [`docs/NAV.md`](../NAV.md) and the round-2 results notes.)

Full guide: **[`docs/NAV.md`](../NAV.md)**.

## Also in 0.7.0

### Verification contracts + the termination gate
A declared definition of *done* the commit gate enforces — exit-0 executable checks plus
tree-pinned human attestations, so "done" can't be claimed without the evidence it named. Includes
locked contracts, sealed held-out checks, `status --json` observability, and a stop-gate driver
protocol. See [`docs/CONTRACTS.md`](../CONTRACTS.md).

### Tamper-evident audit ledger
A compliance control layer that records gate decisions and attestations in a hash-chained ledger
you can verify with `cartogate audit verify`. Honest about its bounds. See
[`docs/AUDIT.md`](../AUDIT.md).

### Devin agent support
`cartogate init --agent devin` joins Cursor, VS Code, Claude Code, and Codex as a first-class
adoption surface. See [`docs/INTEGRATIONS.md`](../INTEGRATIONS.md).

## Upgrading

Cartogate installs from git — a release is a tag plus a `stable` fast-forward, nothing to download.

```bash
pipx install --force "git+https://github.com/cartogate/cartogate.git@stable"
cartogate --version          # 0.7.0

# Navigation is an optional extra (pipx injects the dep directly):
pipx inject cartogate 'playwright>=1.49'          # nav (Playwright adapter)
pipx inject cartogate 'selenium>=4.27'            # nav-selenium (optional)
playwright install chromium                       # fetch a browser once
```

The gate and every existing tool are unchanged and require no extra. Navigation is entirely opt-in.
